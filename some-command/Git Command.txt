GIT CLONE ORIGIN <BRANCH-NAME>: To use the repository on your local machine/system

GIT PULL ORIGIN <BRANCH-NAME>: To get the updated <BRANCH-NAME>

GIT PUSH ORIGIN <BRANCH-NAME>: To push your changes to <BRANCH-NAME> or to update it

GIT ADD <FILE OR DIRECTORY NAME>: Used to add changes to the staging area

GIT ADD .: Used to add changes to all areas

GIT COMMIT -M: Used to record changes in the repository

GIT STATUS: Shows the current status of the repository and any unstaged changes

GIT LOG: History

GIT CHECKOUT <BRANCH-NAME>: To switch branches

GIT BRANCH <NEW-BRANCH>: Create a branch

GIT BRANCH <EXISTING-BRANCH>: List branches

GIT BRANCH -D <EXISTING-BRANCH>: Delete a branch

GIT REMOTE -V: To see which remote is active in the repository

GIT REMOTE SET-URL ORIGIN <REPOSITORY-REMOTE-URL>: To change the remote of the repository

GIT REMOTE ADD: To add a remote to the repository

GIT MERGE <BRANCH-NAME>: To merge changes from a branch where you are. For example, if you have branches named user-info and user-auth and you want to merge or include the updates from those branches into the main branch, first use GIT CHECKOUT ORIGIN MAIN to switch to the main branch, then you can use GIT MERGE <BRANCH-NAME> or GIT MERGE user-info or GIT MERGE user-auth.
